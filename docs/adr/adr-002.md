# CI/CD Self-Hosted com Git Bare Hooks + Docker Compose (FastAPI + React Stack)

Automatiza deploy de Windows → Labserver (Ubuntu 24.04) usando Git hooks, sem GitHub ou DockerHub, com zero downtime manual.

**Referências:**

* [liviaerxin.github.io: CI/CD Githook Docker](https://liviaerxin.github.io/blog/ci-cd-githook-docker)
* [Docker Forums: docker-compose-watch](https://forums.docker.com/t/docker-compose-watch-keeps-cycling-fastapi-app-on-sync-restart/144559)
* [GitHub Actions: Full Stack FastAPI Template](https://github.com/fastapi/full-stack-fastapi-template)

---

## Objetivo

* Replicar ambiente dev Windows → prod auto-deploy na rede local.
* Edita no Windows → `git push lab main` → rebuild e deploy Docker Compose automático no labserver.
* Persistência do banco Postgres e múltiplos serviços (FastAPI, UI React, Admin React).

---

## Pré-requisitos

```
Windows 10/11 dev
Ubuntu 24.04 LTS no labserver (kernel >=6.x)
Docker CE + docker-compose-plugin
Rede local 192.168.18.0/24
Projeto FastAPI + React UI + React Admin + Postgres15
```

## 1. Instalação Base no Labserver

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update && sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin -y
sudo usermod -aG docker $USER && newgrp docker
```

## 2. Transferência Inicial do Projeto (Evitar Conflitos Git)

### ✅ Recomendada: `rsync`

```powershell
# PowerShell
rsync -av --progress C:\Users\bunga\stuffs\queue_system/ bungaantonio@192.168.18.3:/home/bungaantonio/queue_system/
```

> Mantém permissões, evita sobrescrever histórico Git existente, transfere apenas diferenças.

### Alternativa: `git clone` direto

```bash
# Labserver
git clone ssh://bungaantonio@windows_pc_ip/c$/Users/bunga/stuffs/queue_system.git ~/queue_system
```

> Mantém histórico Git e evita divergências com commits iniciais.

⚠️ Evitar `scp -r` direto se houver `.git`, pois pode gerar divergência e precisar de `--force` depois.

## 3. Git Bare Repository + Auto-Deploy Hook

```bash
mkdir -p ~/repos/queue_system.git
cd ~/repos/queue_system.git
git init --bare
```

### Criar hook `post-receive`

```bash
cat > hooks/post-receive << 'EOF'
#!/bin/bash
WORK_TREE=/home/bungaantonio/queue_system
GIT_DIR=/home/bungaantonio/repos/queue_system.git
git --work-tree=$WORK_TREE --git-dir=$GIT_DIR checkout -f main
cd $WORK_TREE

# Docker Compose rebuild & deploy
docker compose down
docker compose build --no-cache
docker compose up -d

# Logging
echo "$(date): Deploy concluído" >> /home/bungaantonio/deploy.log
EOF

chmod +x hooks/post-receive
```

> Agora qualquer push para `lab` dispara rebuild/deploy.

## 4. Inicialização do Working Directory

```bash
cd ~/queue_system
git init
git add .
git commit -m "Initial commit"
git remote add origin ~/repos/queue_system.git

# Primeiro deploy
docker compose up -d --build
```

## 9. Multi-remote e fluxo de deploy

* `origin` → GitHub ou backup
* `lab` → Labserver auto-deploy
* Quando você é a fonte autoritária:

  ```bash
  git push lab main --force
  ```
* PRs externos → `origin`, revisar → push `lab main` para deploy.

## 10. Alternativas e opções avançadas

1. **Docker Compose Watch** (experimental)

```yaml
x-docker-compose-watch:
  - action: sync+restart
    path: ./api
    target: /app
```

* `docker compose watch` em screen/tmux.
* Auto-sync do diretório local (Linux), não recomendado para Windows dev.

2. **Watchtower**

* Atualiza containers via DockerHub/registry.
* Requer imagens públicas ou privadas:

```yaml
services:
  watchtower:
    image: containrrr/watchtower
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    command: --interval 300
```

3. **GitHub Actions (Cloud CI)**

```yaml
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: 192.168.18.3
          username: bungaantonio
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd ~/myapp
            git pull
            docker compose down && docker compose up -d --build
```

> Útil se você quiser CI cloud, mas aqui priorizamos self-hosted.
